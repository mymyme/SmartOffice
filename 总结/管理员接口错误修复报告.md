# 管理员接口错误修复报告

## 问题描述

1. **GET /api/admin/users** 返回 500 错误
2. **POST /api/admin/users** 返回 400 错误

## 问题原因

### 1. 数据库查询结果处理错误

`db.query()` 方法返回的是数组（rows），但代码中错误地使用了数组解构 `[result]`，导致：
- `countResult.total` 访问失败（应该是 `countResult[0].total`）
- `result.insertId` 为 undefined（INSERT 操作需要使用 `db.pool.execute()`）

### 2. LIMIT 和 OFFSET 参数绑定问题

MySQL 的 `LIMIT` 和 `OFFSET` 不能使用参数绑定（`?`），需要直接拼接数值。

## 修复内容

### 1. 修复 User.create() 方法

**修改前**：
```javascript
const result = await db.query(...);
return result.insertId;  // undefined
```

**修改后**：
```javascript
const [result] = await db.pool.execute(...);
return result.insertId;  // 正确获取插入ID
```

### 2. 修复 User.list() 方法

**修改前**：
```javascript
const [countResult] = await db.query(...);
const total = countResult.total;  // 错误

const users = await db.query(..., [...values, limit, offset]);  // LIMIT/OFFSET 参数绑定错误
```

**修改后**：
```javascript
const countResult = await db.query(...);
const total = countResult[0].total;  // 正确

const limitValue = parseInt(limit);
const offsetValue = parseInt(offset);
const users = await db.query(
    `... LIMIT ${limitValue} OFFSET ${offsetValue}`,  // 直接拼接
    values
);
```

### 3. 修复 User.count() 方法

**修改前**：
```javascript
const [result] = await db.query(...);
return result.count;  // 错误
```

**修改后**：
```javascript
const result = await db.query(...);
return result[0].count;  // 正确
```

### 4. 修复 adminController.js 中的查询

修复了所有使用 `db.query()` 的地方：
- `getUserDetail()` - 用户统计信息查询
- `getStats()` - 系统统计数据查询
- `getUserLogs()` - 用户日志查询

**修改模式**：
```javascript
// 修改前
const [result] = await db.query(...);
return result.count;

// 修改后
const result = await db.query(...);
return result[0].count;
```

## 注意事项

1. **SQL 注入防护**：虽然 LIMIT 和 OFFSET 直接拼接，但已通过 `parseInt()` 确保是整数，避免 SQL 注入
2. **数据库查询结果**：`db.query()` 始终返回数组，需要访问第一个元素 `result[0]`
3. **INSERT 操作**：需要使用 `db.pool.execute()` 来获取 `insertId`
