# 用户列表显示为空数组问题修复

## 问题根本原因 🎯

前端发送的API请求包含空字符串参数：
```
GET /api/admin/users?page=1&limit=20&search=&role=&isActive=
```

后端在处理 `isActive` 参数时，代码逻辑为：
```javascript
isActive: isActive !== null ? isActive === 'true' : null
```

当 `isActive=''`（空字符串）时：
- `isActive !== null` 为 `true`（空字符串不是null）
- `isActive === 'true'` 为 `false`（空字符串不等于'true'）
- 最终 `isActiveFilter` 被设置为 `false`，表示只查询**已禁用**的用户
- 由于所有用户都是启用状态（`is_active=1`），查询结果为空数组

## 修复方案 ✅

修改 `backend/controllers/adminController.js` 中的参数处理逻辑：

### 修改前
```javascript
isActive: isActive !== null ? isActive === 'true' : null
```

### 修改后
```javascript
// 处理 isActive 参数：只有当值为 'true' 或 'false' 时才应用过滤
// 空字符串视为不过滤
let isActiveFilter = null;
if (isActive === 'true') {
    isActiveFilter = true;
} else if (isActive === 'false') {
    isActiveFilter = false;
}

// 使用 isActiveFilter 而不是原来的表达式
isActive: isActiveFilter
```

## 修复逻辑说明

| isActive 值 | 原逻辑结果 | 新逻辑结果 | 查询效果 |
|------------|---------|---------|---------|
| `''` (空字符串) | `false` ❌ | `null` ✅ | 不过滤，返回所有用户 |
| `'true'` | `true` ✅ | `true` ✅ | 只返回启用用户 |
| `'false'` | `false` ✅ | `false` ✅ | 只返回禁用用户 |
| `null` | `null` ✅ | `null` ✅ | 不过滤，返回所有用户 |
| `undefined` | `null` ✅ | `null` ✅ | 不过滤，返回所有用户 |

## 问题总结

这是一个经典的**空字符串与布尔值处理**问题：
- 前端表单的空值默认提交为空字符串 `''`
- 后端需要正确区分 `''`（不过滤）、`'true'`（只要启用）、`'false'`（只要禁用）
- 直接使用三元运算符无法正确处理空字符串的情况
- 需要显式判断字符串值，将空字符串转换为 `null`（表示不过滤）
