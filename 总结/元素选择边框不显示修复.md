# 元素选择边框不显示问题修复

## 🐛 问题描述

**用户反馈**: 启用预览编辑后，鼠标悬停或点击元素时，看不到选择边框

---

## 🔍 根本原因

### iframe样式隔离

```
问题链条：
1. CSS样式定义在主页面的 <style> 标签中
   ↓
2. 预览内容显示在 iframe 中
   ↓
3. iframe 有独立的文档对象
   ↓
4. iframe 不会继承主页面的样式
   ↓
5. .preview-element-hover 和 .preview-selection-highlight 样式不生效
   ↓
6. 边框无法显示 ❌
```

### iframe的特性

```javascript
// 主页面的样式
<style>
  .preview-element-hover {
    outline: 2px solid blue;
  }
</style>

// iframe中的HTML
<iframe>
  <div class="preview-element-hover">
    <!-- 这个div看不到蓝色边框！ -->
  </div>
</iframe>
```

**原因**: iframe有独立的`document`对象，不继承父页面样式。

---

## 🔧 解决方案

### 方案：动态注入样式到iframe

在iframe加载完成后，将选择样式动态注入到iframe的`<head>`中。

#### 1. 创建样式注入函数

```javascript
// 注入元素选择样式到iframe
function injectSelectionStylesToIframe(iframeDoc) {
    // 检查是否已经注入（避免重复）
    if (iframeDoc.getElementById('selection-styles-injected')) {
        return;
    }

    // 创建 <style> 元素
    const style = iframeDoc.createElement('style');
    style.id = 'selection-styles-injected';
    style.textContent = `
        /* 元素悬停高亮 */
        .preview-element-hover {
            outline: 2px solid rgba(101, 175, 255, 0.8) !important;
            outline-offset: 0px !important;
            background: rgba(101, 175, 255, 0.1) !important;
            cursor: pointer !important;
            transition: all 0.15s ease !important;
        }

        /* 元素选中高亮 */
        .preview-selection-highlight {
            outline: 2px solid #1a73e8 !important;
            outline-offset: 0px !important;
            background: rgba(26, 115, 232, 0.08) !important;
            position: relative !important;
        }
    `;

    // 添加到iframe的 <head>
    iframeDoc.head.appendChild(style);
    console.log('[样式注入] 元素选择样式已注入到iframe');
}
```

#### 2. 在iframe加载时调用

```javascript
function initPreviewSelection() {
    const previewFrame = document.getElementById('preview-frame');

    previewFrame.addEventListener('load', function () {
        const iframeDoc = previewFrame.contentDocument || previewFrame.contentWindow.document;

        // ✅ 注入元素选择样式到iframe
        injectSelectionStylesToIframe(iframeDoc);

        // 添加事件监听器...
        iframeDoc.addEventListener('mouseover', handleElementHover);
        iframeDoc.addEventListener('click', handleElementClick);
        // ...
    });
}
```

## 🎯 技术细节

### 1. 样式注入时机

```javascript
previewFrame.addEventListener('load', function () {
    // ✅ 在iframe加载完成后立即注入
    injectSelectionStylesToIframe(iframeDoc);
});
```

**为什么在load事件？**
- 确保iframe的document已经创建
- 确保head元素已经存在
- 在用户交互前注入，避免延迟

### 2. 防止重复注入

```javascript
if (iframeDoc.getElementById('selection-styles-injected')) {
    return; // ✅ 已经注入过，跳过
}
```

**为什么需要检查？**
- 避免重复创建style元素
- 提高性能
- 保持DOM整洁

### 3. 使用 !important

```css
outline: 2px solid rgba(101, 175, 255, 0.8) !important;
```

**为什么使用 !important？**
- 确保样式优先级最高
- 覆盖用户代码中的可能冲突样式
- 保证选择效果始终可见

### 4. 创建独立的style元素

```javascript
const style = iframeDoc.createElement('style');
style.id = 'selection-styles-injected';
iframeDoc.head.appendChild(style);
```

**为什么不直接修改已有样式？**
- 不影响用户的原有代码
- 易于识别和管理
- 可以单独删除或更新

## 💡 扩展知识

### iframe样式隔离的其他解决方案

#### 方案1：CSS Variables（不适用本场景）
```css
/* 主页面 */
:root {
  --hover-color: blue;
}

/* iframe会继承CSS变量吗？ */
/* 答：不会，除非同源并且显式设置 */
```

#### 方案2：Link外部样式表
```javascript
const link = iframeDoc.createElement('link');
link.rel = 'stylesheet';
link.href = '/styles/selection.css';
iframeDoc.head.appendChild(link);
```

**缺点**：需要额外的HTTP请求

#### 方案3：动态注入（本项目采用）✅
```javascript
const style = iframeDoc.createElement('style');
style.textContent = '...';
iframeDoc.head.appendChild(style);
```

**优点**：
- 无需额外请求
- 即时生效
- 易于维护
