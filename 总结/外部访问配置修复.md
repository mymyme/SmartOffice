# 外部访问配置修复

## 问题描述

前端通过外部IP（`http://8.152.98.33:3005`）访问时，调用后端API（`http://8.152.98.33:3006/api`）一直显示"加载中"，请求超时。

## 根本原因

后端服务器使用默认的 `app.listen(PORT)` 启动，未显式指定监听地址。在某些环境下，这可能导致服务器仅监听 IPv6 地址（`:::3006`）或本地回环地址，无法从外部IP访问。

## 监听地址说明

| 监听地址 | 格式 | 访问范围 |
|---------|------|---------|
| `localhost` | `127.0.0.1:3006` | ❌ 仅本机可访问 |
| `127.0.0.1` | `127.0.0.1:3006` | ❌ 仅本机可访问 |
| 默认（未指定） | `:::3006` 或 `*:3006` | ⚠️ 依赖系统配置，可能无法外部访问 |
| `0.0.0.0` | `0.0.0.0:3006` | ✅ 所有网络接口，可外部访问 |

## 修复方案 ✅

修改 `backend/server.js` 中的 `app.listen()` 调用，显式指定监听所有网络接口（`0.0.0.0`）：

### 修改前
```javascript
app.listen(PORT, () => {
    console.log(`服务器启动在端口 ${PORT}`);
});
```

### 修改后
```javascript
app.listen(PORT, '0.0.0.0', () => {
    console.log(`   端口: ${PORT}`);
    console.log(`   地址: http://localhost:${PORT}`);
    console.log(`   外部访问: http://0.0.0.0:${PORT}`);
});
```

## 安全注意事项

### 1. 防火墙配置
确保服务器防火墙允许以下端口：
```bash
# 允许3005端口（前端）
sudo ufw allow 3005/tcp

# 允许3006端口（后端API）
sudo ufw allow 3006/tcp

# 不要开放3307端口（MySQL应该仅内部访问）
```

### 2. 生产环境建议
- 使用 Nginx 作为反向代理
- 配置 HTTPS/SSL 证书
- 使用域名代替IP地址
- 将前后端统一到同一域名下（避免CORS问题）
- MySQL仅监听 `127.0.0.1`，不对外开放

### 3. Nginx 配置示例
```nginx
server {
    listen 80;
    server_name yourdomain.com;

    # 前端静态文件
    location / {
        proxy_pass http://127.0.0.1:3005;
    }

    # 后端API
    location /api/ {
        proxy_pass http://127.0.0.1:3006;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 问题总结

这是一个**网络监听配置**问题：
- Node.js 的 `app.listen(port)` 在不同环境下行为不一致
- 必须显式指定 `0.0.0.0` 才能确保监听所有网络接口
- 在容器化环境（Docker）中尤其重要
- 生产环境应配合反向代理和防火墙使用
