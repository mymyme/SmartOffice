# AI局部编辑功能优化说明

## 🔧 优化内容

### 1. 增强了选中内容验证
**问题**: AI回复"我没有看到您选中的内容"
**原因**: 发送请求时`selectionText`可能为空
**优化**:
- 添加发送前验证
- 如果内容为空，显示友好的错误提示
- 添加控制台日志便于调试

### 2. 优化了AI提示词
**修改前**:
```
我选中了以下内容：

${selectionText}

请根据以下指令修改：${instruction}

请直接返回修改后的HTML代码，不要解释。
```

**修改后**:
```
我选中了以下HTML内容：

${selectionText}

请根据以下指令修改上述HTML：${instruction}

要求：
1. 只返回修改后的完整HTML代码
2. 保持原有的class和结构
3. 不要添加任何解释文字
4. 确保HTML格式正确
```

### 3. 改进了选中内容显示
**修改前**: 纯文本显示，格式混乱
**修改后**: 使用`<pre>`标签，HTML转义，格式清晰

### 4. 添加了调试日志
- 打开对话框时记录选中内容
- 发送请求时记录完整prompt
- 便于排查问题

## 🎯 正确使用方法

### 步骤1: 清除浏览器缓存
```
重要！必须先清除缓存才能加载新代码
按 Ctrl+Shift+R 强制刷新
```

### 步骤2: 启用预览编辑
```
点击工具栏的 "🖊 启用预览编辑" 按钮
```

### 步骤3: 选中要编辑的元素
有两种方式：

#### 方式1: 点击选中（推荐）
```
1. 在预览区点击任意元素
2. 元素会被蓝色虚线框高亮
3. 浮动工具栏出现
4. 点击 "✏️ AI编辑" 按钮
```

#### 方式2: 右键快捷方式（最快）
```
1. 在预览区右键点击任意元素
2. 自动高亮并打开局部编辑对话框
```

### 步骤4: 检查选中内容
```
对话框顶部会显示选中的HTML代码
✅ 如果能看到HTML代码 → 正常
❌ 如果显示"请在预览区选中要编辑的内容" → 重新选择
```

### 步骤5: 输入编辑指令
常用指令示例：

#### 样式修改
```
- "添加红色背景"
- "改为蓝色文字"
- "字体加粗"
- "增加内边距20px"
- "添加边框"
```

#### 内容修改
```
- "改写为更专业的表述"
- "翻译成英文"
- "简化文字"
- "添加图标"
```

#### 布局修改
```
- "改为flex布局"
- "居中对齐"
- "改为两列布局"
```

### 步骤6: 发送并查看预览
```
1. 点击"发送"按钮
2. 等待AI生成修改方案
3. 查看预览效果
4. 满意则点击"应用修改"
5. 不满意则点击"拒绝"，重新输入指令
```

## 🔍 调试方法

如果遇到"我没有看到您选中的内容"错误：

### 1. 打开浏览器控制台
```
按 F12
切换到 Console 标签
```

### 2. 查看日志
查找以下日志：

#### 打开对话框时
```
打开局部编辑对话框，选中内容: <div class="...">...</div>
```
✅ 如果看到完整HTML → 选中正常
❌ 如果看到空字符串 → 选中失败

#### 发送请求时
```
发送给AI的prompt: 我选中了以下HTML内容：...
```
✅ 如果包含HTML代码 → 正常
❌ 如果内容为空 → 选中内容丢失

### 3. 常见问题排查

#### 问题1: 选中内容为空
**可能原因**:
- 在iframe加载完成前就选中了元素
- 页面刚刷新，事件监听器还未初始化

**解决方法**:
```
1. 等待页面完全加载
2. 重新点击预览编辑按钮
3. 重新选中元素
```

#### 问题2: 选中后内容丢失
**可能原因**:
- 选中后点击了其他地方
- 触发了清除选中的事件

**解决方法**:
```
1. 右键点击元素（直接打开对话框）
2. 不要点击对话框外的区域
3. 立即输入指令并发送
```

#### 问题3: AI回复格式错误
**可能原因**:
- AI返回了带解释的文本
- 代码提取逻辑失败

**解决方法**:
```
1. 查看控制台的prompt日志
2. 确认指令清晰明确
3. 重新发送更具体的指令
```

## 💡 最佳实践

### 1. 指令要具体明确
```
❌ 不好: "改一下"
✅ 好: "将背景色改为红色，字体改为白色"

❌ 不好: "美化"
✅ 好: "添加阴影效果，圆角8px，内边距16px"
```

### 2. 一次只修改一个方面
```
✅ 推荐顺序:
1. 先修改颜色 → 应用
2. 再修改布局 → 应用
3. 最后修改内容 → 应用

❌ 避免:
一次性要求多个复杂修改
```

### 3. 保持简单的HTML结构
```
✅ 适合局部编辑:
- 单个div
- 标题文字
- 按钮元素
- 列表项

⚠️ 不太适合:
- 整页的HTML
- 复杂的嵌套结构
- 包含脚本的代码
```

### 4. 利用预览功能
```
✅ 好习惯:
- 应用前先查看预览
- 不满意就拒绝重试
- 满意后再应用

❌ 坏习惯:
- 不看预览直接应用
- 应用后发现问题再撤销
```

## 🎨 示例演示

### 示例1: 修改颜色

**选中内容**:
```html
<div class="text-gray-400">排放强度</div>
```

**指令**: `将文字改为红色`

**AI返回**:
```html
<div class="text-red-500">排放强度</div>
```

---

### 示例2: 添加背景

**选中内容**:
```html
<div class="flex justify-between">
    <span class="text-gray-400">排放强度</span>
    <span class="text-white">0.92 kg/万元</span>
</div>
```

**指令**: `添加红色背景，文字改为白色`

**AI返回**:
```html
<div class="flex justify-between bg-red-500 p-4">
    <span class="text-white">排放强度</span>
    <span class="text-white">0.92 kg/万元</span>
</div>
```

---

### 示例3: 修改布局

**选中内容**:
```html
<div>
    <span>标题</span>
    <span>内容</span>
</div>
```

**指令**: `改为垂直布局，元素之间增加间距`

**AI返回**:
```html
<div class="flex flex-col gap-4">
    <span>标题</span>
    <span>内容</span>
</div>
```

## ⚡ 快捷技巧

### 1. 键盘快捷键
```
Enter: 发送指令
Shift+Enter: 换行（多行指令）
Esc: 关闭对话框
```

### 2. 连续编辑
```
选中元素后，可以连续发送多条指令：
1. "改为红色" → 应用
2. "加粗" → 应用
3. "增加边距" → 应用

无需重新选择元素
```

### 3. 使用浏览器控制台
```
在控制台输入:
selectionText

可以查看当前选中的内容
```
