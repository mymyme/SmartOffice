# 问题解决记录

## 问题描述

**错误信息**：
```
visual-editor.html:1132  POST http://localhost:3006/api/auth/login net::ERR_CONNECTION_REFUSED
登录错误: TypeError: Failed to fetch
```

## 问题分析

### 1. 后端服务器未启动
- 前端页面 `visual-editor.html` 尝试连接 `http://localhost:3006/api/auth/login`
- 后端服务器未运行，导致连接被拒绝

### 2. bcrypt 模块问题（已解决）
- **错误**：`Error: /media/storage/project/aicode/backend/node_modules/bcrypt/lib/binding/napi-v3/bcrypt_lib.node: invalid ELF header`
- **原因**：bcrypt 模块的二进制文件与当前系统架构不兼容
- **解决方案**：使用 `bcryptjs` 替代 `bcrypt`
  - `bcryptjs` 是纯 JavaScript 实现，无需编译
  - API 完全兼容，只需修改 require 语句

### 3. 数据库连接问题（待解决）
- **错误**：`❌ 数据库连接失败: connect ECONNREFUSED 127.0.0.1:3307`
- **原因**：MySQL 可能未运行在 3307 端口
- **配置要求**：项目要求 MySQL 运行在 3307 端口

## 已完成的修复

### ✅ 修复 bcrypt 模块
1. 卸载 `bcrypt` 模块
2. 安装 `bcryptjs` 模块
3. 修改 `backend/utils/hash.js`：
   ```javascript
   // 修改前
   const bcrypt = require('bcrypt');

   // 修改后
   const bcrypt = require('bcryptjs');
   ```

## 待解决的问题

### ⚠️ MySQL 数据库连接

**问题**：MySQL 未运行在 3307 端口

**解决方案选项**：

#### 方案1：配置 MySQL 运行在 3307 端口（推荐）
```bash
# 编辑 MySQL 配置文件
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

# 添加或修改以下行
port = 3307

# 重启 MySQL
sudo systemctl restart mysql
```

#### 方案2：修改项目配置使用 3306 端口（临时方案）
如果 MySQL 已经在 3306 端口运行，可以修改配置：

1. 修改 `database/config.js`：
   ```javascript
   port: process.env.DB_PORT || 3306,  // 改为 3306
   ```

2. 或者在 `backend/.env` 中设置：
   ```env
   DB_PORT=3306
   ```

#### 方案3：使用环境变量覆盖
创建或修改 `backend/.env` 文件：
```env
DB_HOST=localhost
DB_PORT=3306  # 或 3307，根据实际 MySQL 端口
DB_USER=monaco_user
DB_PASSWORD=monaco_password_2024
DB_NAME=monaco_editor_db
```
