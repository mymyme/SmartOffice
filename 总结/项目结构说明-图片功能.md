### 二、后端文件

#### 1. images.js (新文件)
**路径**: `/backend/routes/images.js`

**功能**: 图片管理API路由

**主要接口**:
```javascript
// 上传图片
POST /api/images/upload
- 接收multipart/form-data
- 验证用户身份
- 保存到user_id目录
- 生成缩略图
- 返回图片信息

// 获取图片列表
GET /api/images/list
- 仅返回当前用户的图片
- 支持分页和搜索
- 返回缩略图URL

// 获取单个图片
GET /api/images/:id
- 验证图片归属
- 返回完整信息

// 删除图片
DELETE /api/images/:id
- 验证所有权
- 删除文件和数据库记录
```

**依赖**:
- `multer` - 文件上传处理
- `sharp` - 图片处理和缩略图生成
- `crypto` - 文件名hash
- `fs.promises` - 文件系统操作

**安全特性**:
- JWT身份验证
- user_id强制过滤
- 文件类型白名单
- 文件大小限制
- 防目录遍历

**代码量**: ~350行

### 三、数据库文件

#### 1. 003_create_images_table.sql (新文件)
**路径**: `/database/migrations/003_create_images_table.sql`

**内容**: 创建images表的SQL语句

**表结构**:
```sql
CREATE TABLE images (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,              -- 关联用户
  filename VARCHAR(255),              -- 存储文件名
  original_name VARCHAR(255),         -- 原始文件名
  file_size INT,                      -- 文件大小
  mime_type VARCHAR(50),              -- MIME类型
  width INT,                          -- 宽度
  height INT,                         -- 高度
  storage_path VARCHAR(500),          -- 存储路径
  url VARCHAR(500),                   -- 访问URL
  thumbnail_url VARCHAR(500),         -- 缩略图URL
  usage_count INT DEFAULT 0,          -- 使用次数
  created_at TIMESTAMP,               -- 创建时间
  updated_at TIMESTAMP,               -- 更新时间

  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  INDEX idx_user_id (user_id)
);
```

**特点**:
- 外键关联users表
- 级联删除
- 按user_id索引

## 🔗 文件依赖关系

```
visual-editor.html
    ↓ (HTTP请求)
server.js → imageRoutes (images.js)
    ↓ (数据库查询)
database (images表)
    ↓ (文件操作)
uploads/ (图片存储)
```

### 请求流程

```
用户上传图片
    ↓
前端 FormData → POST /api/images/upload
    ↓
authMiddleware (验证JWT)
    ↓
multer (接收文件)
    ↓
sharp (生成缩略图)
    ↓
数据库 (保存记录)
    ↓
返回图片URL
    ↓
前端显示图片
```

## 📝 维护说明

### 日常维护

#### 1. 清理过期图片
```sql
-- 删除30天未使用的图片
DELETE FROM images
WHERE usage_count = 0
  AND created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);
```

#### 2. 检查存储空间
```bash
# 查看uploads目录大小
du -sh /media/storage/project/aicode/uploads
```

#### 3. 备份图片
```bash
# 备份整个uploads目录
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz uploads/
```

### 故障排查

#### 图片上传失败
1. 检查后端服务是否运行
2. 检查数据库连接
3. 查看后端日志
4. 检查uploads目录权限

#### 图片无法显示
1. 检查静态文件服务配置
2. 验证图片URL路径
3. 检查CORS配置
4. 查看浏览器控制台错误
