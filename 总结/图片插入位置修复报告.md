# 图片插入位置修复报告

## 🐛 问题描述

**用户反馈**: 图片插入的位置不对，默认都插入在最顶部，而不是光标选中的地方

**问题原因**: 打开图片库对话框后，`selectedRange`失效，导致无法在正确位置插入

---

## 🔍 根本原因分析

### Range失效的过程

```
1. 用户在预览区选中文字或光标位置
   → selectedRange 被保存

2. 点击浮动工具栏的"插入图片"按钮
   → 打开图片库对话框

3. 对话框获得焦点
   → iframe失去焦点
   → Selection对象被清除
   → Range对象失效 ❌

4. 用户选择图片，点击"插入图片"
   → 尝试使用 selectedRange
   → Range已失效，无法插入到正确位置
   → 代码fallback到插入到编辑器末尾
```

### 为什么会失效？

**浏览器行为**:
- 当焦点从iframe转移到主页面（对话框）时
- iframe的Selection会被自动清除
- Range对象失去了关联的DOM节点
- 调用Range.insertNode()会抛出异常

**原有代码的问题**:
```javascript
// ❌ 问题代码
function insertImageAtSelection(imageHtml) {
    // 直接使用selectedRange，但它可能已经失效
    if (selectedRange) {
        selectedRange.insertNode(imgElement);
    }
}
```

---

## 🔧 修复方案

### 方案: 保存Range快照

在打开对话框**之前**保存Range，在插入时使用保存的Range。

#### 修改1: 添加savedRange变量

```javascript
// ========== 图片上传功能 ==========
let selectedImageData = null;
let currentImageContext = 'ai-chat';
let savedRange = null; // ✅ 新增：保存打开对话框前的Range
```

#### 修改2: 打开对话框时保存Range

```javascript
function openImageInsertAtSelection() {
    if (!selectedRange) {
        alert('请先选中要插入图片的位置');
        return;
    }

    // ✅ 保存当前的Range（打开对话框后会失效）
    savedRange = selectedRange;
    console.log('[图片插入] 已保存选中位置:', savedRange);

    // 设置上下文为选中位置插入
    currentImageContext = 'selection';

    // 打开图片上传对话框
    const modal = document.getElementById('image-modal');
    modal.classList.add('show');
}
```

#### 修改3: 插入时使用保存的Range

```javascript
function insertImageAtSelection(imageHtml) {
    // ✅ 优先使用保存的Range
    const rangeToUse = savedRange || selectedRange;

    if (!rangeToUse) {
        console.log('选中位置已失效，将插入到编辑器末尾');
        insertImageToEditor(imageHtml);
        return;
    }

    try {
        const previewFrame = document.getElementById('preview-frame');
        const iframeDoc = previewFrame.contentDocument || previewFrame.contentWindow.document;

        // 创建图片元素
        const tempDiv = iframeDoc.createElement('div');
        tempDiv.innerHTML = imageHtml;
        const imgElement = tempDiv.firstElementChild;

        console.log('[图片插入] 使用Range插入:', rangeToUse);

        // ✅ 使用保存的Range插入
        try {
            // 删除选中内容（如果有的话）
            if (!rangeToUse.collapsed) {
                rangeToUse.deleteContents();
            }
            rangeToUse.insertNode(imgElement);
            console.log('✅ 图片已插入到选中位置');
        } catch (e) {
            console.warn('Range插入失败，尝试使用selectedElement:', e);
            // 如果Range失效，尝试使用selectedElement作为备用
            if (selectedElement && selectedElement.parentNode) {
                selectedElement.parentNode.insertBefore(imgElement, selectedElement.nextSibling);
                console.log('✅ 图片已插入到选中元素后面');
            } else {
                throw new Error('Range和Element都失效');
            }
        }

        // 同步到编辑器
        syncPreviewToCode();
        makeImageResizable(imgElement);

        // ✅ 清除保存的Range
        savedRange = null;

        updateStatus('✅ 图片已插入成功');
    } catch (error) {
        console.error('选中位置插入失败:', error);
        insertImageToEditor(imageHtml);
        savedRange = null;
    }
}
```

## 💡 技术要点

### Range对象的特性

#### 1. Range会失效的情况
```javascript
// 失效场景：
- 焦点转移（iframe → 主页面）
- DOM节点被删除
- 页面重新加载
- Selection被清除
```

#### 2. 保存Range的方法
```javascript
// ✅ 方法1：直接引用（本次采用）
savedRange = selectedRange;

// ✅ 方法2：克隆Range
savedRange = selectedRange.cloneRange();

// ⚠️ 方法3：保存位置信息
savedRangeInfo = {
    startContainer: range.startContainer,
    startOffset: range.startOffset,
    endContainer: range.endContainer,
    endOffset: range.endOffset
};
// 缺点：节点引用可能失效
```

### Range.insertNode的行为

```javascript
// collapsed = true（光标位置）
range.insertNode(node);
// → 在光标位置插入节点

// collapsed = false（选中了内容）
range.deleteContents(); // 先删除选中内容
range.insertNode(node); // 再插入节点
// → 替换选中内容
```
