# 登录历史接口500错误修复报告

## 问题描述

个人中心的登录历史功能报错：
```
GET http://8.152.98.33:8301/api/user/login-history?limit=50 500 (Internal Server Error)
```

## 问题分析

### 根本原因

MySQL预处理语句（Prepared Statements）不支持将`LIMIT`和`OFFSET`作为参数绑定。

### 错误代码

在`userController.js`和`adminController.js`中，以下代码会导致500错误：

```javascript
// 错误的写法
const logs = await db.query(
    `SELECT ... FROM user_logs
     WHERE user_id = ?
     LIMIT ? OFFSET ?`,
    [userId, limit, offset]  // ❌ LIMIT和OFFSET不能作为参数绑定
);
```

### 错误信息

```
Incorrect arguments to mysqld_stmt_execute
```

这是因为MySQL的预处理语句机制限制了某些子句（如`LIMIT`和`OFFSET`）不能使用参数绑定。

## 修复方案

### 修复原理

将`LIMIT`和`OFFSET`的值直接拼接到SQL字符串中，而不是作为参数绑定。由于这些值已经通过`parseInt()`处理过，所以是安全的，不存在SQL注入风险。

### 修复代码

#### 1. 修复 `backend/controllers/userController.js`

**修改位置：** 第324-331行，`getLoginHistory`函数

**修改前：**
```javascript
const logs = await db.query(
    `SELECT action, description, status, ip_address, user_agent, created_at
     FROM ${db.tables.userLogs}
     WHERE user_id = ? AND action IN ('LOGIN_SUCCESS', 'LOGIN_FAILED', 'LOGOUT')
     ORDER BY created_at DESC
     LIMIT ? OFFSET ?`,
    [req.userId, limit, offset]
);
```

**修改后：**
```javascript
const logs = await db.query(
    `SELECT action, description, status, ip_address, user_agent, created_at
     FROM ${db.tables.userLogs}
     WHERE user_id = ? AND action IN ('LOGIN_SUCCESS', 'LOGIN_FAILED', 'LOGOUT')
     ORDER BY created_at DESC
     LIMIT ${limit} OFFSET ${offset}`,
    [req.userId]
);
```

**同时修复：** 第335-340行，`count`查询的解构方式

**修改前：**
```javascript

    `SELECT COUNT(*) as total ...`,
    [req.userId]
);
// 访问: countResult.total
```

**修改后：**
```javascript

    `SELECT COUNT(*) as total ...`,
    [req.userId]
);
// 访问: countResult[0].total
```

## 安全性说明

虽然使用了字符串拼接，但这种做法是安全的，原因如下：

1. **参数验证**：`limit`和`offset`都经过`parseInt()`处理
2. **类型保证**：`parseInt()`确保返回的是数字或`NaN`
3. **默认值**：使用`|| 运算符`提供默认值，确保始终是有效数字
4. **无用户输入**：这些值来自查询参数，经过严格验证

```javascript
const page = parseInt(req.query.page) || 1;      // 默认1
const limit = parseInt(req.query.limit) || 20;   // 默认20
const offset = (page - 1) * limit;               // 计算得出
```

### 2. 数据库查询封装

考虑封装常用的分页查询模式：

```javascript
// utils/dbHelper.js
async function queryWithPagination(sql, params, page, limit) {
    const offset = (page - 1) * limit;
    const paginatedSql = `${sql} LIMIT ${limit} OFFSET ${offset}`;
    return await db.query(paginatedSql, params);
}
```
