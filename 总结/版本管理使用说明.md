## 开发者操作

### 发布新版本时的操作步骤

#### 1. 修改版本号

需要修改以下4个文件中的 `APP_VERSION`：

**visual-editor.html (第1952行)**
```javascript
const APP_VERSION = '2.0.1'; // 改为新版本号，如 '2.0.2'
```

**login.html (第443行)**
```javascript
const APP_VERSION = '2.0.1'; // 改为新版本号
```

**admin.html (第639行)**
```javascript
const APP_VERSION = '2.0.1'; // 改为新版本号
```

**profile.html (第574行)**
```javascript
const APP_VERSION = '2.0.1'; // 改为新版本号
```

#### 2. 版本号规则

```
格式：主版本.次版本.修订号
示例：2.0.1
     │ │ └─ 修订号：Bug修复、小优化
     │ └─── 次版本：新功能、新页面
     └───── 主版本：重大变更、架构升级
```

**何时增加版本号：**
- 主版本 (X.0.0)：重大架构变更、不兼容的API变更
- 次版本 (2.X.0)：新增功能、新增页面
- 修订号 (2.0.X)：Bug修复、样式调整、文案优化

#### 3. 提交代码

```bash
# 修改版本号后提交
git add .
git commit -m "chore: 升级版本到 v2.0.2"
git tag v2.0.2
git push origin main --tags
```

## 功能说明

### 自动检测机制

**首次访问：**
```
用户访问 → 未发现版本记录 → 保存当前版本 → 正常使用
```

**版本未变化：**
```
用户访问 → 版本号匹配 → 无需操作 → 正常使用
```

**版本已更新：**
```
用户访问 → 检测到新版本 → 清除缓存 → 保留登录信息 → 显示通知 → 正常使用
```

### 数据保护

**自动保留：**
- ✅ 登录Token
- ✅ 用户信息

**自动清除：**
- ❌ 所有其他缓存数据
- ❌ 草稿内容（注意提醒用户保存）
- ❌ 临时数据

## 测试方法

### 1. 模拟版本更新

在浏览器控制台执行：

```javascript
// 设置旧版本
localStorage.setItem('app_version', '2.0.0');

// 刷新页面
location.reload();

// 观察结果：
// - 控制台应显示版本更新日志
// - visual-editor.html 应显示更新通知
// - localStorage 应清除（除token和user）
// - 新版本号应保存为 2.0.1
```

### 2. 检查版本信息

在浏览器控制台执行：

```javascript
// 查看当前版本（仅visual-editor.html）
getVersionInfo();

// 查看localStorage中的版本
localStorage.getItem('app_version');
```

### 3. 验证缓存清除

```javascript
// 1. 设置测试数据
localStorage.setItem('test_data', 'should be cleared');

// 2. 修改版本号触发更新
localStorage.setItem('app_version', '1.0.0');

// 3. 刷新页面
location.reload();

// 4. 检查数据是否清除
localStorage.getItem('test_data'); // 应返回 null
```

## 常见问题

### Q1: 用户看不到新功能？

**A:** 可能原因和解决方法：

1. **浏览器强缓存HTML文件**
   - 解决：Ctrl+Shift+R 强制刷新

2. **CDN或Nginx缓存**
   - 解决：清除服务器缓存

3. **用户未访问页面**
   - 解决：提示用户访问新页面

### Q2: 版本更新后用户登录状态丢失？

**A:** 不应该发生，检查：
- `checkVersion()` 函数是否正确保存token
- token是否过期
- API是否正常

### Q3: 如何让所有用户立即更新？

**A:** 无法强制，但可以：
1. 发送通知提醒用户刷新
2. 在页面上显示"有新版本"提示
3. 实现自动刷新功能

### Q4: 需要保留草稿数据怎么办？

**A:** 修改 `checkVersion()` 函数：

```javascript
// 保存草稿
const draft = localStorage.getItem('draft');

// 清除localStorage
localStorage.clear();

// 恢复草稿
if (draft) localStorage.setItem('draft', draft);
```

## 注意事项

### ⚠️ 重要提醒

1. **版本号必须统一**
   - 所有4个页面的版本号必须相同
   - 否则会导致用户在不同页面看到不同行为

2. **测试后再发布**
   - 修改版本号后务必测试
   - 确保缓存清除正常
   - 确保登录状态保留

3. **提醒用户保存**
   - 更新前提醒用户保存草稿
   - 或修改代码保留草稿数据

4. **服务器缓存**
   - 确保服务器不缓存HTML文件
   - 已添加Meta标签，但可能不够
   - 建议配置Nginx/Apache
