# 404 页面问题解决报告

## 问题描述

点击菜单中的"个人中心"、"管理后台"、"退出登录"都提示 404 错误：
```
404 - 页面未找到
请求的页面不存在
```

## 问题原因

前端服务器 (`frontend/services/server.js`) 只配置了部分路径别名映射：
- ✅ `/visual-editor.html` → `/demos/visual-editor.html`
- ✅ `/default-code-in-editor.html` → `/demos/default-code-in-editor.html`
- ❌ `/profile.html` → 未配置
- ❌ `/admin.html` → 未配置
- ❌ `/login.html` → 未配置

菜单链接使用的路径：
- `/profile.html` - 个人中心
- `/admin.html` - 管理后台
- `/login.html` - 登录页面（退出登录后跳转）

这些路径没有映射到实际文件位置 `/demos/` 目录，导致返回 404 错误。

## 解决方案

### 修改文件
`frontend/services/server.js`

### 添加路径别名映射

在路径别名映射部分添加了以下配置：

```javascript
// 路径别名映射
if (pathname === '/visual-editor.html') {
    pathname = '/demos/visual-editor.html';
}
if (pathname === '/default-code-in-editor.html') {
    pathname = '/demos/default-code-in-editor.html';
}
// 新增的路径映射
if (pathname === '/profile.html') {
    pathname = '/demos/profile.html';
}
if (pathname === '/admin.html') {
    pathname = '/demos/admin.html';
}
if (pathname === '/login.html') {
    pathname = '/demos/login.html';
}
```

## 验证结果

### 1. 个人中心页面
```bash
curl http://localhost:3005/profile.html
```
✅ 返回页面内容，标题为"个人中心 - Monaco Editor"

### 2. 管理后台页面
```bash
curl http://localhost:3005/admin.html
```
✅ 返回页面内容，标题为"管理员后台 - Monaco Editor"

### 3. 登录页面
```bash
curl http://localhost:3005/login.html
```
✅ 返回页面内容，标题为"登录 - Monaco Editor"

## 功能验证

### 菜单链接
- ✅ **个人中心** (`/profile.html`) - 正常跳转
- ✅ **管理后台** (`/admin.html`) - 正常跳转（仅管理员可见）
- ✅ **退出登录** - 调用 `logout()` 函数，跳转到 `/login.html`

### 退出登录功能

退出登录函数 (`logout()`) 的功能：
1. 调用后端 API `/api/auth/logout` 登出
2. 清除本地存储的 token 和 user 信息
3. 跳转到 `/login.html` 页面

3. **文件位置**：所有页面文件都在 `frontend/demos/` 目录下
