# 创建用户 400 错误修复报告

## 问题描述

前端调用 `POST /api/admin/users` 创建用户时返回 400 (Bad Request) 错误。

## 问题原因

前端在创建用户时发送了不必要或格式不正确的字段：

1. **发送了 `is_active` 字段**：创建用户时不应该发送此字段，因为新用户默认是激活状态
2. **发送了空字符串的 `full_name`**：空字符串可能导致验证问题，应该发送 `null` 或不发送该字段

## 修复方案

修改 `frontend/demos/admin.html` 中的 `submitUser()` 方法，在创建用户时只发送必需的字段：

### 修改前
```javascript
const body = this.modalMode === 'create' ? this.userForm : { ... };
```

### 修改后
```javascript
let body;
if (this.modalMode === 'create') {
    // 创建用户时，只发送必需字段，过滤空值
    body = {
        username: this.userForm.username,
        email: this.userForm.email,
        password: this.userForm.password,
        role: this.userForm.role
    };
    // 如果 full_name 不为空，才添加
    if (this.userForm.full_name && this.userForm.full_name.trim()) {
        body.full_name = this.userForm.full_name.trim();
    }
} else {
    // 编辑用户时
    body = {
        email: this.userForm.email,
        full_name: this.userForm.full_name || null,
        role: this.userForm.role,
        is_active: this.userForm.is_active
    };
}
```

## 修复内容

1. **创建用户时**：
   - 只发送必需字段：`username`, `email`, `password`, `role`
   - 如果 `full_name` 有值且不为空，才添加到请求体中
   - 不发送 `is_active` 字段（由后端设置默认值）

2. **编辑用户时**：
   - 保持原有逻辑不变
   - 发送 `email`, `full_name`, `role`, `is_active` 字段

## 注意事项

1. **字段过滤**：创建用户时只发送后端验证规则要求的字段
2. **空值处理**：空字符串应该转换为 `null` 或不发送
3. **数据清理**：使用 `trim()` 去除前后空格
