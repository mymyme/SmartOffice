### 三、数据库

#### 1. 数据表结构
**表名**: `images`

```sql
CREATE TABLE images (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,           -- 用户ID（关键隔离字段）
  filename VARCHAR(255),           -- 存储文件名
  original_name VARCHAR(255),      -- 原始文件名
  file_size INT,                   -- 文件大小
  mime_type VARCHAR(50),           -- 文件类型
  width INT,                       -- 图片宽度
  height INT,                      -- 图片高度
  storage_path VARCHAR(500),       -- 存储路径
  url VARCHAR(500),                -- 访问URL
  thumbnail_url VARCHAR(500),      -- 缩略图URL
  usage_count INT DEFAULT 0,       -- 使用次数
  created_at TIMESTAMP,            -- 创建时间
  updated_at TIMESTAMP,            -- 更新时间
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  INDEX idx_user_id (user_id)     -- 查询优化索引
);
```

**特点**:
- ✅ 外键级联删除（用户删除时自动删除其图片）
- ✅ 按用户ID索引优化查询
- ✅ 支持缩略图URL
- ✅ 记录图片使用次数

### 六、功能演示流程

#### 场景1：AI智能插入图片
```
用户操作：
1. 点击AI对话框的 [📎 上传图片] 按钮
2. 选择本地图片文件
3. 等待上传完成（显示进度条）
4. 设置图片宽度、对齐方式、Alt文字
5. 点击 [插入图片] 按钮

系统行为：
→ 上传到服务器（按用户ID存储）
→ 自动生成缩略图
→ 保存到数据库
→ 发送消息给AI："我上传了一张图片..."
→ AI分析并生成包含图片的HTML代码
→ 自动插入到编辑器
→ 实时预览区显示图片
```

#### 场景2：从图片库选择
```
用户操作：
1. 点击AI对话框的 [🎨 我的图片库] 按钮
2. 查看已上传的所有图片（仅自己的）
3. 点击选择一张图片
4. 设置插入选项
5. 点击 [插入图片] 按钮

系统行为：
→ 加载用户图片库（仅user_id匹配的）
→ 显示缩略图网格
→ 用户选择图片
→ 发送给AI处理
→ 插入到编辑器
→ 预览区显示
```

#### 场景3：使用外部URL
```
用户操作：
1. 切换到 [图片URL] Tab
2. 输入图片URL地址
3. 点击 [预览] 按钮确认图片可用
4. 设置插入选项
5. 点击 [插入图片] 按钮

系统行为：
→ 验证URL有效性
→ 预览图片
→ 生成HTML代码
→ 插入到编辑器
→ 预览区显示（外部链接）
```
