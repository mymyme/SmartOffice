### 重启 FRP 客户端

FRP客户端由root运行，需要手动重启以应用新的8301端口配置。

**选择以下任一方法：**

#### 方法1：使用 systemctl（推荐）
```bash
sudo systemctl restart frpc
```

#### 方法2：重载配置
```bash
sudo kill -USR1 3113670
```

#### 方法3：手动重启
```bash
# 停止
sudo kill 3113670

# 启动
sudo /home/bn/project/frp/frpc -c /home/bn/project/frp/frpc.toml &
```

重启后，检查8301端口是否监听：

## 💡 技术要点总结

### 1. 参数处理
空字符串需要显式判断，不能依赖三元运算符的隐式转换。

### 2. 网络监听
Express应用在生产环境必须显式绑定 `0.0.0.0` 以支持外部访问。

### 3. FRP端口映射
前后端必须使用不同的公网端口，前端通过检测访问端口自动切换API端口。

### 4. 跨域配置
CORS配置应支持正则匹配，允许同一IP的不同端口访问。

### 5. 调试日志
在关键位置添加 `console.log` 输出，方便问题定位。
