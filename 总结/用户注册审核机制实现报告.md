# 用户注册审核机制实现报告

## 功能概述

实现了用户注册后需要管理员审核激活的机制，新注册用户默认为禁用状态，需要管理员在后台开通权限后才能登录使用。

## 实现时间

- 实现时间：2025-11-17
- 状态：✅ 已完成

## 需求背景

为了加强用户管理和系统安全，新注册的用户不应立即获得系统访问权限，需要经过管理员审核后才能使用系统功能。

## 技术实现

### 1. 数据库层修改

**文件：** `backend/models/User.js`

**修改内容：** 在 `create` 方法中添加 `is_active` 字段支持

```javascript
static async create(userData) {
    const { username, email, password, role = 'user', full_name, is_active } = userData;
    const password_hash = await hashPassword(password);

    // 如果没有明确指定 is_active，默认为 0（禁用）
    const activeStatus = is_active !== undefined ? is_active : 0;

    const [result] = await db.pool.execute(
        `INSERT INTO ${db.tables.users}
         (username, email, password_hash, role, full_name, is_active, created_at)
         VALUES (?, ?, ?, ?, ?, ?, NOW())`,
        [username, email, password_hash, role, full_name || null, activeStatus]
    );

    return result.insertId;
}
```

**关键点：**
- ✅ 默认 `is_active = 0`（禁用状态）
- ✅ 支持手动指定 `is_active`（管理员创建用户时可设为启用）
- ✅ 向后兼容，不影响现有功能

### 2. 注册接口修改

**文件：** `backend/controllers/authController.js`

**修改前：**
```javascript
// 创建用户
const userId = await User.create({
    username,
    email,
    password,
    role: 'user'
});

// 自动登录...
const tokenResponse = jwtUtil.createTokenResponse(newUser);
// 返回token，用户可以直接登录
```

**修改后：**
```javascript
// 创建用户（默认禁用状态）
const userId = await User.create({
    username,
    email,
    password,
    role: 'user',
    is_active: 0  // 新注册用户默认禁用
});

// 记录注册日志
await logUserAction(
    userId,
    'REGISTER_SUCCESS',
    '用户注册成功，等待管理员审核',
    'success',
    ipAddress,
    userAgent
);

// 不自动登录，直接返回成功消息
return res.status(201).json({
    success: true,
    message: '恭喜注册成功，请联系管理员开通权限',
    data: {
        username,
        email,
        needActivation: true
    }
});
```

**关键变化：**
- ❌ 移除自动登录逻辑
- ❌ 不再生成 token 和 session
- ❌ 不再设置 cookie
- ✅ 明确设置 `is_active: 0`
- ✅ 返回需要激活的提示信息

```
   ↓
3. 查看待审核用户（is_active = 0）
   ↓
4. 点击「编辑」按钮
   ↓
5. 勾选「启用状态」复选框
   ↓
6. 保存修改
   ↓
7. 用户可以正常登录
```

## 数据库字段

### users 表

|-------|------|--------|------|
| is_active | TINYINT(1) | 1 | 用户状态：1=启用，0=禁用 |

- 新注册：`is_active = 0`
- 管理员激活后：`is_active = 1`
- 管理员禁用后：`is_active = 0`
